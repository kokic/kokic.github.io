前置内容

笛卡尔积

定义 设 $A$ 和 $B$ 是两个集合，称集合 $\{(x,y)|x\in A\land y\in B \}$ 为 $A$ 和 $B$ 的 笛卡尔积，也称 直积，记为 $A\times B$，即 $A\times B\overset{\text{def}}{=}\{(x,y)|x\in A\land y\in B \}$。特别的，将 $A\times\cdots\times A$（$n$ 个 $A$）记为 $A^n$。
一般地说，笛卡尔积运算不满足交换律，即 $A\times B\neq B\times A$（当 $A\neq\varnothing\land B\neq\varnothing\land A\neq B$ 时）
笛卡尔积运算不满足结合律，即 $(A\times B)\times C\neq A\times(B\times C)$（当 $A\neq\varnothing\land B\neq\varnothing\land C\neq\varnothing$ 时）
笛卡尔积运算对并和交运算满足分配律，即
$A\times(B\cup C)=(A\times B)\cup(A\times C)$,$(B\cup C)\times A=(B\times A)\cup(C\times A)$，
$A\times(B\cap C)=(A\times B)\cap(A\times C)$,$(B\cap C)\times A=(B\times A)\cap(C\times A)$。

附：更一般的情况是对于 $n$ 个集合的直积，$A_1\times\cdots\times A_n\overset{\text{def}}{=}\{(a_1,\cdots,a_n)|\displaystyle\bigwedge_{k=1}^na_k\in A_k\}$，或者更进一步的 $\displaystyle\prod_{k\in K}A_k\overset{\text{def}}{=}\{(a_k)_{k\in K}|\bigwedge_{k\in K}a_k\in A_k\}$。

合成

将映射 $S^2\to S$ 称为 $S$ 的合成，具体而言，$S$ 上的合成是将 $S$ 中的元素 $a,b$ 结合成另一个元素 $t$。合成作用在 $(a,b)$ 上得到 $t$ 常记为 $a\cdot b=t$，这里的 $\cdot$ 并非是常规意义上的数的乘法。
合成的结合律是指 $\forall a,b,c\in S,(a\cdot b)\cdot c=a\cdot(b\cdot c)$，合成的交换律是指 $\forall a,b\in S,a\cdot b=b\cdot a$，自然，结合律是比交换律更基础的性质，这一点从函数的复合上可以看出。
设 $A$ 是一个集合，$f$ 和 $g$ 均为 $A\to A$ 的函数，令 $g\cdot f$ 表示复合映射 $a\leadsto g(f(a))$，设 $h$ 也是一个 $A\to A$ 的函数，函数的复合满足结合律，也即 $(h\cdot g)\cdot f=h\cdot(g\cdot f)$。

基于此，我们可以在 javascript 中定义 compose 方法：

//  compose :: func -> func -> a -> a
var compose = f => g => x => f(g(x));

即 $\text{compose}(f,g)=f\cdot g$，同样的可定义：

var f = a => `f(${a})`, g = a => `g(${a})`, h = a => `h(${a})`;

发现 compose(compose(h)(g))(f)('x') 与 compose(h)(compose(g)(f))('x') 的结果均为 "h(g(f(x)))"，也就是说 

compose(compose(h)(g))(f)('x') == compose(h)(compose(g)(f))('x')

总是为真。

关系

定义 设 $A,B,T$ 均是非空集合，则 $A\times B$ 到 $T$ 的一个映射 $f$，称为 $A$ 和 $B$ 到 $T$ 的一个 代数运算。因而我们知道，若有 $a\in A,b\in B$，则 $(a,b)\in A\times B,f((a,b))=t\in T$，即 $a$ 与 $b$ 的运算结果是 $t$，常记 $f((a,b))$ 为 $a\cdot b$，则 $a\cdot b=t$。

定义 设 $A\neq\varnothing$，$R$ 是 $A^2$ 的一个子集，$a,b\in A$，若 $(a,b)\in R$，则称 $a$ 与 $b$ 有关系 $R$，记为 $aRb$，且称 $R$ 为 $A$ 的一个 二元关系，不致混淆的情况下，也记为 $a\sim b$，若既想使用 $\sim$ 又希望不同的关系有所区别，则也有 $a\overset{R}{\sim}b$ 这一记法。

等价类

定义 若集合 $A$ 中的一个二元关系 $R$ 满足：
 自反性：$\forall a\in A,aRa$
 对称性：$\forall a,b\in A,aRb\Rarr bRa$
 传递性：$\forall a,b,c\in A,(aRb)\land(bRc)\Rarr aRc$
则称关系 $R$ 为 $A$ 的一个 等价关系，所有与 $a$ 等价的元素构成的集合，称作以 $a$ 为代表的 等价类，记为 $\bar{a}$。

考虑不全为零的复数对 $(z_1,z_2)$ 与 $(w_1,w_2)$，若 $\exist p\in\mathbb{C}\land p\neq 0$ 使 $z_1=pw_1,z_2=pw_2$，则 $(z_1,z_2)$ 与 $(w_1,w_2)$ 等价，这个例子出自华罗庚《高等数学引论》第三册第一章第三节，用于定义一维射影空间，在此不过多叙述，但在后面的内容将会继续引用这本书中的部分例子。

$A$ 关于 $R$ 的等价类也可记作 $[a]_R$。当只考虑一个关系时，我们省去下标 $R$ 并把这个等价类写作 $[a]$，这也就是说 $[a]\overset{\text{def}}{=}\{b\in A:bRa\}$。
设 $R$ 是一个等价关系，若 $(a,b)\in R$ ，则称 $a$ 等价于 $b$，记作 $a\equiv b$。
$A$ 中所有的等价类构成的集合 $\{\bar{a}\}$称为 $A$ 对 $R$ 的 商集合，记为 $A/R$，即 $A/R\overset{\text{def}}{=}\{[a]:a\in A\}$。

初等数论中，根据整数模 $n$ 所得的余数把整数分为 $n$ 个等价类，模 $n$ 等价类包含的整数 $a$ 为 $[a]_n=\{a+kn:k\in\mathbb{Z}\}$，所有这样的等价类的集合为 $\mathbb{Z}_n=\{[a]_n:0\le a\le n-1\}$。

同余类

继续回顾初等数论中的一些内容，容易联想到同余的概念。设 $m$ 为正整数，整数 $a$ 与 $b$ 之差 可被 $m$ 整除时，称为 $a$ 和 $b$ 关于模 $m$ 同余，记为 $a\equiv b(\text{mod }m)$。
应知 $a\equiv b(\text{mod }m)$ 的充分必要条件为 $a\text{ mod }m=b\text{ mod }m$，即 $a$ 和 $b$ 被 $m$ 除的余数相等。
亦知其有自反性：$a\equiv a(\text{mod }m)$、对称性：若 $a\equiv b(\text{mod }m)$，则 $b\equiv a(\text{mod }m)$、传递性：若 $a\equiv b(\text{mod }m)$ 且 $b\equiv c(\text{mod }m)$，则 $a\equiv c(\text{mod }m)$，其中 $c$ 为整数。
以及此关系在二元运算 $\pm$ 和 $\times$ 下依然被保持，这也就是说，若 $a\equiv b(\text{mod }m)$ 且 $c\equiv d(\text{mod }m)$，则 $a\pm c\equiv b\pm d(\text{mod }m),ac\equiv bd(\text{mod }m)$，其中 $d$ 为整数，于是便可十分自然的引出更一般的关系。
$$

定义 设集合 $A$ 中有一种二元运算 “$\cdot$”，如果 $A$ 的一个等价关系 $R$ 在该运算下依然被保持，即 $\forall a,b,c,d \in A,aRb,cRd\Rarr(a\cdot c)R(b\cdot d)$，则称 $R$ 为 $A$ 中关于 “$\cdot$” 的一个 同余关系，此时 $\bar{a}$ 被称作 同余类。

gcd

有关 $\text{mod}$ 的应用，最为熟悉的应当是 欧几里德算法（辗转相除法），其用于计算两个非负整数 $a,b$ 的最大公约数。这一算法的原理为 $\text{gcd}(a,b)=\text{gcd}(b,a\text{ mod }b)$，其中 $a\ge b,b\neq0$，即两个整数的最大公约数等于其中较小的那个数和两数相除余数的最大公约数。

var gcd_euclid = (a, b) => b == 0 ? a : gcd_euclid(b, a % b);

又 $\forall a,b\in\mathbb{Z},\exist x,y\in\mathbb{Z}$ 使 $ax+by=\text{gcd}(a,b)$，由此便有了 扩展欧几里得算法，在计算 $\text{gcd}(a,b)$ 的同时还能找到 $x,y$。

var gcd_euclid_extended = (a, b) => 
  b == 0 ? [a, 1, 0] : 
    (([d, x, y]) => [d, y, x - Math.floor(a / b) * y])
    (gcd_euclid_extended(b, a % b));

不难看出，gcd_euclid_extended 返回一组数 $(d,x,y)$，其中 $d=\text{gcd}(a,b)$。

借助此法，便能很容易的求解关于 $x$ 的模线性方程 $ax\equiv b(\text{mod }n),n\gt 0$。

function modular_linear_eqs(a, b, n) {
  var [d, x, y] = gcd_euclid_extended(a, n);
  var res = new Array();
  if (b % d == 0) { // d | b
    x = (x * b / d) % n;
    if (x < 0) x = (x + n) % n; // x > 0
    for (var i = 0; i < d; i ++)
      res.push((x + i * (n / d)) % n);
  }
  return res;
}

补充：假设方程 $ax\equiv b(\text{mod }n)$ 有解，$x_0$ 是方程的任意一解，则方程对模 $n$ 恰有 $d$ 个不同的解，分别为：$x_k=x_0+\dfrac{nk}{d}$, 其中 $k\in\mathbb{Z}\land 0\le k\le d-1$。

群

定义 设 $G$ 是一个带有二元运算 “$\cdot$” 的非空集合，即 $(a,b)\mapsto a\cdot b:G^2\to G$，且其中的运算满足
 封闭律：$\forall a,b\in G,a\cdot b\in G$
 结合律：$\forall a,b,c\in G,(a\cdot b)\cdot c=a\cdot(b\cdot c)$
 有幺元：$\exists e\in G$，使 $\forall a\in G$，有 $e\cdot a=a\cdot e=a$，称 $e$ 为幺元（单位元）
 有逆元：$\forall a\in G$，$\exists b\in G$，使 $b\cdot a=a\cdot b=e$，称 $b$ 为 $a$ 的逆元，记为 $a^{-1}$
则称 $(G,\cdot)$ 是一个 群，也可简记为 $G$，无歧义时，$a\cdot b$ 可写成 $ab$，自然，群是有一个二元运算的代数结构。

因而，群 $G$ 的运算满足左消去律和右消去律，即 $\forall a,b,c\in G,ab=ac\Rarr b=c,ba=ca\Rarr b=c$，群 $G$ 中的幺元是唯一的（这由幺元的存在性推出），任一元 $a$ 的逆元 $a^{-1}$ 是唯一的（这由逆元的存在性推出）。在群 $G$ 中，$\forall a,b\in G$，方程 $ax=b$ 及 $xa=b$ 的解存在且唯一。

阿贝尔群

定义 若一个群 $(G,\cdot)$ 满足交换律，即 $\forall a,b\in G$，总有 $a\cdot b=b\cdot a$，则称群 $G$ 为 交换群（阿贝尔群），也称为 加法群，这是因为习惯上，一个交换群 $G$ 的代数运算符号用 “+” 表示。
如无说明，则默认为非交换群。非交换群也称为 乘法群，乘群的运算符号可以用 “$\cdot$”（也常用 “$*$”）表示或者省略不写。

群同构

设 $f:A\to B$，若 $f(A)=B$，则 $f$ 是满射，若 $f(a_1)\neq f(a_2)\forall (a_1,a_2\in A\land a_1\neq a_2)$，则 $f$ 是单射。

定义 设 $(G,\cdot)$ 与 $(G^{\prime},\cdot^{\prime})$ 都是群，$f:G\to G^{\prime}$，若 $\forall a,b\in G$，都有 $f(a\cdot b)=f(a)\cdot^{\prime}f(b)$，则称 $f$ 是 $G$ 到 $G^{\prime}$ 的一个同态映射，称 $G$ 与 $G^{\prime}$ 同态，可以看出，群同态 $f$ 保持运算。

注：将 $f(a\cdot b)=f(a)\cdot^{\prime}f(b)$ 简写成 $f(ab)=f(a)f(b)$ 并不会产生歧义，一种直观的理解是，分别对 $G$ 和 $G^{\prime}$ 重载了运算符 *，$ab$ 调用的是 $G$ 中的 *，而 $f(a)f(b)$ 调用的则是 $G^{\prime}$ 中的 *，关于什么是运算符重载，可以参见 Classes (II) - C++ Tutorials 。

特别的，当 $G^{\prime}=G$，即 $f:G\to G$ 时，称 $f$ 为（群的）自同态映射，或者更直接的，称一个到自身的映射 $f:G\to G$ 为 $G$ 的 自同态，以 $\text{End}(G)$ 表示所有 $G$ 的自同态的集合，即 $\text{End}(G)\overset{\text{def}}{=}\text{Hom}(G,G)$。
依映射中单射和满射的概念，便有单同态和满同态，而当同态 $f$ 为双射（既单又满）时，则称 $f$ 是 $G$ 到 $G^{\prime}$ 的一个同构映射，称 $G$ 与 $G^{\prime}$ 同构，记作 $G\cong G^{\prime}$，同样的，当$G\cong G^{\prime}\land G^{\prime}=G$ 时，称 $f$ 为 $G$ 的 自同构，以 $\text{Aut}(G)$ 表示所有 $G$ 的自同构的集合，即 $\text{Aut}(G)\overset{\text{def}}{=}\text{Aut}(G,G)$。

附：一般以 $A\hookrightarrow B$ 表示 $A$ 到 $B$ 的单射，以 $A\twoheadrightarrow B$ 表示 $A$ 到 $B$ 的满射，以 $A\overset{\sim}{\rarr}B$ 表示 $A$ 到 $B$ 的双射。$\text{Hom}(X,Y)$ 表示所有从 $X$ 到 $Y$ 的态射（此处即为群同态）的集合。

幺半群

定义 设 $S$ 是一个带有运算 “$\cdot$” 的非空集合，且其中的运算满足
 封闭律：$\forall a,b\in S,a\cdot b\in S$
 结合律：$\forall a,b,c\in S,(a\cdot b)\cdot c=a\cdot(b\cdot c)$
则称 $(S,\cdot)$ 是一个 半群，若半群 $S$ 还有幺元，则称 $S$ 为 幺半群，即 $(S,\,\cdot\,,e)$。
若运算 $\cdot$ 还满足交换律，则 $(S,\,\cdot\,,e)$ 称为 交换幺半群（阿贝尔幺半群），交换幺半群中的运算常写成 “$+$”，即 $(S,+,e)$。

可知，若半群 $S$ 满足 $\forall a,b\in S$，方程 $ax=b,xa=b$ 均有解，则 $S$ 是群。

Julia 允许使用 $\ast$ 来连接字符串文本。尽管对于提供 $+$ 函数用于字符串拼接的语言使用者而言，$*$ 似乎是一个令人惊讶的选择，但 $\ast$ 的这种用法在数学中早有先例，尤其是在抽象代数中。

在数学上，$+$ 通常表示可交换运算——运算对象的顺序不重要。一个例子是矩阵加法：对于任何形状相同的矩阵 $A$ 和 $B$，都有 $A+B=B+A$。与之相反，$\ast$ 通常表示不可交换运算——运算对象的顺序很重要。例如，对于矩阵乘法，一般 $AB\neq BA$。同矩阵乘法类似，字符串拼接是不可交换的。在这一点上，对于插入字符串的拼接操作，$\ast$ 是一个自然而然的选择，与它在数学中的用法一致。

更确切地说，有限长度字符串集合 $S$ 和字符串拼接操作 $\ast$ 构成了一个自由幺半群 $(S,\ast)$。该集合的单位元是空字符串 “”。当一个自由幺半群不是交换的时，它的运算通常表示为 $\cdot$ 或 $\ast$，或者类似的符号，而非暗示交换性的 $+$。

引用自 字符串 · Julia 中文文档 。

子群

定义 设 $H$ 是群 $G$ 的一个非空子集，如果对于 $G$ 的运算 $H$ 也成群，则称 $H$ 为 $G$ 的一个 子群，记作 $H\le G$，当 $H\neq G$，即 $H\lt G$ 时，$H$ 被称为 真子群。

例如，所有的线性变换成群，所有的整线性变换也成群，为线性变换群的子群（华罗庚《高等数学引论》第三册第一章第四节）。
