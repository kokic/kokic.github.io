I’ve been busy configuring my work environment on a new device Redmi Book Pro 15, 2022 recently and encountered many (almost serious) problems during this period (about three weeks, of course, linux only. The default system of the device is Windows 11, works great). Here are some more details on why I use linux and choose Manjaro, and the price to pay for it (common sense: Linux linux usually requires additional settings on some special devices, so unfortunately I’m not among the “lucky few” ). To save time for someone with the same problem, I’ll list the solutions first.

  • USB autosuspend - https://wiki.archlinux.org/title/Power_management#USB_autosuspend. autosuspend can be disabled by setting power/autosuspend (usbcore.autosuspend) to -1 i.e., never autosuspend.

  • Screen corruption after resuming from suspend or hibernation (7) & System does not return from suspend (25) see NVIDIA/Troubleshooting.

$\kern{2em}\mathcal{Q}_1$ How is this device? $\leadsto$ CPU - Intel i7 12650H (A piece of news: Even though my old device was using an “extremely old” 3rd gen Intel i7 processor (which seemed top hardware back then) and it has been used for about 8 years, with a great SSD it’s still possible to do most of the work, until recently I was still using it for programming in Java / Haskell / OCaml etc. on Windows 10), 2.3GHz, Graphics - NVIDIA GeForce RTX2050 (I’m not sure if this is the main cause of the problem, but in most cases NVIDIA’s graphics do cause a lot of trouble (again, only for linux. It’s not that NVIDIA’s products have any quality problems, in fact it’s in machine learning field is quite outstanding, not to mention cuda)), RAM - 8GB $\times$ 2 LDDR5 5200MHz, Hard Disk - SSD 512GB … Everything looks fine, it’s thin and light and has great performance, a great high-res screen. In the early years this only happened on the Mac Book Pro. Of course none of this is free, the whole device ended up costing me $1000, but it’s still a lot cheaper than a Mac Book Pro (half the price for a Mac Book Pro of the same size).

$\kern{2em}\mathcal{Q}_2$ What’s the problem? $\leadsto$ A lot of times I’ve programmed on Windows, but this time I’ve changed my mind, mostly because of the language I’m using (if you’ve tried Haskell on Windows you’ll understand, for every GHC - Glasgow Haskell Compiler, ghcup will all install a whole set of separate MSYS2, which leads me to believe that Haskeller has found a technique to produce a completely free unlimited capacity hard drive) and tools have better support on Linux (package management).
